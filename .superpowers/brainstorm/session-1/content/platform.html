TUI Terminal for Lattice: Architecture Options

Select the stack and approach you prefer for the Lattice Terminal UI

A

Python + Textual

A modern, reactive TUI framework for Python. Provides rich widgets, CSS grid layouts, scrolling, mouse support, and a beautiful terminal design system. Needs python environment with textual installed.

B

Node.js + Ink (React in Terminal)

Uses React to build interactive command-line apps. Excellent component architecture and reactive state management. Needs Node.js runtime and dependency installation.

C

Python + Rich (Lightweight Console)

Uses Python's rich library with standard Prompt and layout panels. Minimalistic, fast loading, no heavy event-loop required, and runs easily on top of existing python scripts.

Trade-offs & Considerations

Pros of Option A (Textual)

  • Incredibly premium design with widgets, panels, and sidebars out-of-the-box.
  • Full mouse support, scrolling, and keyboard-driven navigation.
  • Easy integration with python-based codebase analysis.

Cons of Option A (Textual)

  • Requires installing textual package in the environment.
  • Slightly higher memory footprint.
