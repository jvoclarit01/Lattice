# Figures and Tables Domain

Figures and tables best practices for academic papers.

## When to Activate

Activate when creating figures, designing tables, or visualizing data.

## Core Principles

1. **Be Clear** — Make figures and tables clear
2. **Be Accurate** — Ensure accuracy
3. **Be Consistent** — Use consistent style
4. **Be Informative** — Make them informative

## Figure Design

### Types of Figures

**Line plots:**
- Time series data
- Trends over time
- Continuous variables
- Multiple series comparison

**Bar charts:**
- Categorical data
- Comparisons across groups
- Discrete categories
- Ranked data

**Scatter plots:**
- Relationship between variables
- Correlation visualization
- Distribution patterns
- Outlier identification

**Heatmaps:**
- Matrix data visualization
- Correlation matrices
- Spatial data
- Intensity patterns

**Diagrams:**
- System architecture
- Process flows
- Conceptual models
- Methodology illustrations

**Box plots:**
- Distribution summary
- Outlier detection
- Group comparisons
- Statistical summaries

**Example:**
> **Choosing the right figure type:**
> - Time series → Line plot
> - Category comparison → Bar chart
> - Variable relationship → Scatter plot
> - Distribution → Box plot or histogram

### Figure Elements

**Clear labels:**
- Axis labels with units
- Tick labels
- Data labels
- Legend entries

**Legends:**
- Clear, concise
- Positioned appropriately
- Consistent ordering
- Avoid redundancy

**Axis labels:**
- Descriptive labels
- Include units
- Appropriate font size
- Clear orientation

**Captions:**
- Descriptive and informative
- Explain what is shown
- Note significance
- Provide context

**Example:**
> **Figure caption:**
> "Figure 1: Performance comparison of transformer models on medical text classification. Error bars show 95% confidence intervals. Our proposed method (blue) achieves 94.2% F1 score, outperforming BERT (red) by 12.3 percentage points (p < 0.001)."

### Figure Style

**Use appropriate colors:**
- Colorblind-friendly palettes
- Sufficient contrast
- Meaningful color coding
- Consistent color scheme

**Use appropriate scales:**
- Linear vs logarithmic
- Appropriate range
- Clear tick marks
- Consistent scales across figures

**Use appropriate fonts:**
- Readable size
- Consistent style
- Sans-serif for figures
- Appropriate weight

**Follow journal guidelines:**
- Resolution requirements
- File format specifications
- Size limitations
- Style requirements

**Example:**
> **Color palette recommendations:**
> - Colorblind-friendly: Viridis, Plasma, Cividis
> - High contrast: Black/white with gray
> - Categorical: Set1, Set2, Set3 (ColorBrewer)
> - Sequential: Blues, Greens, Oranges

## Table Design

### Table Elements

**Clear headers:**
- Descriptive column headers
- Appropriate capitalization
- Clear hierarchy
- Consistent formatting

**Clear units:**
- Units in headers or cells
- Consistent notation
- Clear abbreviations
- Standard units

**Clear formatting:**
- Consistent alignment
- Appropriate spacing
- Clear borders
- Readable font size

**Clear captions:**
- Descriptive and informative
- Explain what is shown
- Note significance
- Provide context

**Example:**
> **Table caption:**
> "Table 1: Performance comparison of different models on the medical text classification task. All results are reported as mean ± standard deviation over 5-fold cross-validation. Bold indicates best performance."

### Table Style

**Use consistent formatting:**
- Same font throughout
- Consistent alignment
- Consistent spacing
- Consistent borders

**Use appropriate precision:**
- Significant figures
- Consistent decimal places
- Meaningful precision
- Avoid excessive decimals

**Use appropriate alignment:**
- Numbers: Right-aligned or decimal-aligned
- Text: Left-aligned
- Headers: Centered or left-aligned
- Consistent within columns

**Follow journal guidelines:**
- Table format requirements
- Placement specifications
- Size limitations
- Style requirements

**Example:**
> **Table formatting:**
> - Numbers: Right-aligned, 2 decimal places
> - Text: Left-aligned, sentence case
> - Headers: Centered, title case
> - Borders: Top and bottom only

## Common Mistakes to Avoid

### Figure Mistakes

1. **Unclear labels** — Use descriptive labels with units
2. **Poor color choices** — Use colorblind-friendly palettes
3. **Inappropriate scales** — Choose scales that show data clearly
4. **Missing legends** — Include clear, informative legends
5. **Low resolution** — Use high-resolution figures
6. **Cluttered design** — Keep design clean and simple
7. **Inconsistent style** — Use consistent style across figures
8. **Missing captions** — Include descriptive captions

### Table Mistakes

1. **Unclear headers** — Use descriptive column headers
2. **Inconsistent alignment** — Align numbers and text appropriately
3. **Excessive precision** — Use meaningful precision
4. **Missing units** — Include units in headers
5. **Poor formatting** — Use consistent formatting
6. **Missing captions** — Include descriptive captions
7. **Too much data** — Summarize or split large tables
8. **Unclear abbreviations** — Define abbreviations

## Figure and Table Templates

### Figure Caption Template

```
Figure [number]: [Brief description of what the figure shows]. [Additional details about data, methods, or conditions]. [Key observations or findings]. [Statistical significance if applicable]. [Connection to research question].
```

**Example:**
> Figure 1: Performance comparison of transformer models on medical text classification. Results show mean F1 scores with 95% confidence intervals over 5-fold cross-validation. Our proposed method (blue) achieves 94.2% F1, outperforming BERT (red) by 12.3 percentage points (p < 0.001). This demonstrates the value of domain-specific adaptation for medical NLP tasks.

### Table Caption Template

```
Table [number]: [Brief description of what the table shows]. [Additional details about data, methods, or conditions]. [Key observations or findings]. [Statistical significance if applicable]. [Notes on abbreviations or symbols].
```

**Example:**
> Table 1: Performance comparison of different models on the medical text classification task. All results are reported as mean ± standard deviation over 5-fold cross-validation. Bold indicates best performance. * indicates statistical significance compared to baseline (p < 0.05). N/A indicates not applicable.
